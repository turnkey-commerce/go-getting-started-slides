Forking a Package For Contributions - Issue

    • Internal package names in import referencing wrong location
    • e.g. Assuming forked to github.com/your-username/xo, the original references will be:
      • import (
        "github.com/knq/xo/internal"   "github.com/knq/xo/models"
      )
    • Renaming all of them to github.com/your-username/xo is not practical and error prone

Forking a Package For Contributions - Technique

    1. Go get the package as normal
    2. Fork package as usual on Github
    3. At "go get" code location set fork as remote
      4. git remote add fork https://github.com/your-username/repo.git
    4. Push commits back to fork
    5. Issue pull request from fork
